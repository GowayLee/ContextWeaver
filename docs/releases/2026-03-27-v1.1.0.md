# ContextWeaver v1.1.0

发布日期：2026-03-27

## 发布摘要

本次版本重点补齐了项目级索引配置能力。现在可以在仓库根目录提交 `cwconfig.json`，把索引范围和排除规则作为项目自身的一部分进行管理，而不是依赖用户本地环境变量。

## 重点更新

- 新增项目级索引配置文件 `cwconfig.json`
- 支持 `indexing.includePatterns` 先限定索引范围
- 支持 `indexing.ignorePatterns` 再从范围内剔除路径
- 内置默认排除规则和项目根 `.gitignore` 仍然生效
- 移除旧的 `IGNORE_PATTERNS` 环境变量，不保留向后兼容

## 配置示例

```json
{
  "indexing": {
    "includePatterns": ["src/**", "packages/*/src/**"],
    "ignorePatterns": ["**/generated/**", "**/__snapshots__/**"]
  }
}
```

## 行为说明

- 过滤顺序：内置默认排除 -> `includePatterns` -> `ignorePatterns` -> `.gitignore` -> 扩展名白名单
- `includePatterns` 省略时表示整个仓库都可参与索引
- `includePatterns` 为空数组时表示空范围
- 仓库根的 `cwconfig.json` 自身不会被索引
- 嵌套目录中的同名文件不会被特殊排除
- 配置不支持 `!` negation 模式；`cwconfig.json` 非法时会直接中止索引

## 迁移说明

- 如果你之前依赖 `IGNORE_PATTERNS`，请迁移到仓库根的 `cwconfig.json`
- 旧环境变量已移除，不再读取，也不再文档化
